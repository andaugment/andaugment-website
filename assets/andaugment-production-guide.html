ハーモニー言語 — コード分析

核心原則：低音域で長3度を使わない。 低域の積み重ねた3度は周波数干渉を起こす。代わりにsus2/sus4コードとオープンインターバルを多用。
手法具体例 (Eマイナー)効果
IVmaj7 / IVsus2Amaj7 / Asus2解決感なく浮遊
I/V (第2転回形)Em/B不安定な着地、緊張維持
VIsus2Csus2感情的な頂点
Vsus4 → VBsus4 → Bテンション→リリース
m7コード多用Em7, Am7, Dm7トーナルの曖昧さ

グリッチシンセの再現手法

AoT「Apetitan」などのグリッチ効果（Omnisphere「Rad Drug」+「Power Blowout」使用）の再現：

  • コムフィルターに自動化（周波数/レゾナンスをLFOで動かす）
  • ゲートオートメーション（音量の矩形波LFO）
  • Izotope Stutter Edit 2、Sugar Bytes Effectrix、Sugar Bytes Looperatorが代替

コーラス/ボーカルの使い方

SATBコーラステクスチャー

8Dio LacrimosaとRequiemを重ねる。コーラスはDROPの後にのみ登場させる（ビルドアップ中は不在または強くフィルター）。登場自体が構造的な事件。

nZkボーカル（フォネティック）

意味のある英語/ドイツ語の単語に聞こえるが完全には理解できないシラブルを使う。「感情的フォネティクス」——意味論的ではなく感情的。

ドイツ語の合唱：ヨーロッパの宗教音楽の権威感を付与（"Vogel im Käfig"等）。

金管ヴォイシング（4声構成例 — Eマイナー）

下から上: Tuba (E1) → Bass Trombone (B2) → Tenor Trombones (E3/G3) → Horns (B3/E4) → Trumpets (E5) // 第5音をバストロンボーンに置く → 安定感 // 3度(G)はテナートロンボーン → 低音域に置かない // ホルンは octave reinforcement // トランペットが上部のエネルギーを担う

打楽器レイヤー（3層構成）

レイヤー楽器役割
低音域（基盤）オーケストラバスドラム、タイコ、ティンパニ各小節の1拍目に主打。タイコは16分音符パターン（グリッドより5-10ms遅め）
中音域（リズム）スネア（軍楽風）、ロートタム、フレームドラム半音符または4分音符。セクション間のフィル
高音域（輝き）クラッシュシンバル、サスペンドシンバルスウェルフレーズ頭（毎4〜8小節）のみ。毎拍は使わない
タイミングの秘訣: 打楽器を75%クォンタイズ（完全グリッドではない）にすると人間らしく、物理的に感じられる。特にタイコは5-10msグリッドより遅らせると「呼吸」する。

テンションとリリースの6技法

#技法実装
1和声テンション維持強拍にI（根位置）を置かない。i/V（第2転回）で着地を先延ばし
2オーケストラ密度増加弦→低金管（8小節後）→ホルン（12小節）→トランペット+合唱（コーラス）
3DROP前のネガティブスペース1小節（または1拍）の沈黙がコーラス再登場を最大化
4見かけ上のテンポ倍加クライマックスで弦オスティナートを8分→16分に切替。BPMは変えない
5転調（半音〜全音上）最終コーラス/クライマックスで半音〜全音転調。Em→Fmが即効性高い
6ベースドロップブリッジでベースラインを8小節消す。クライマックスで戻ってきた時の物理的インパクトが倍増

旋律作成ルール

「1回聴いたら歌える」が合格基準。

  • ペンタトニックでフックを作る — Eマイナーペンタ: E-G-A-B-D。最大5音で主题を構築
  • 第6音か第7音でピーク — EからC(♭6)への上昇アーク = 悲しみ+壮大さ
  • 「da-da-DUM」リズムパターン — 8分×2＋4分。記憶に残る最基本リズムセル
  • コールアンドレスポンス — 4小節の金管コール→弦のレスポンス。対話があると単調でなくなる
  • シンコペーション = 危険のシグナル — オフビートアクセントで敵の強大さを表現（シモムラ・植松流）

3Cubase プロフェッショナルテンプレート設計

調査ソース: VI-Control Orchestral Template スレッド群、Steinbergフォーラム、Art Conductor (BabylonWaves)公式、Negative Track Delay Database (VI-Control)
重要原則: テンプレート構築に2〜3日以上かけるな。コアスケルトン（5楽器）を作って実際に曲を書き始める。実際の制作から必要なものが見えてくる。

推奨Cubaseバージョン: Cubase Pro 14

Cubase 14でCPU処理効率が大幅改善（VI-Controlユーザー報告: ピークCPU 25%→12-15%）。大規模テンプレートでの差が顕著。

必須機能（4つ）

VCAフェーダー

グループトラックより優れている理由: 相対的で非破壊的。個別トラックのオートメーションを壊さずにセクション全体のレベルを動かせる。入れ子にできる（楽器VCA → セクションVCA → マスターVCA）。

Visibility Configurations

200トラックテンプレートで正気を保つ唯一の方法。「Strings Only」「Full Orch Mix」「Shorts Only」等の表示設定をキーボードショートカットに登録。

Negative Track Delay

Kontaktライブラリには「音が鳴り始めるまでの時間」がある。インスペクターのDelayフィールドにマイナス値を設定してグリッドに合わせる。VI-Control Negative Track Delay Database Spreadsheetが標準値を網羅している。

Disabled Tracks（デフォルト無効化）

全トラックをデフォルトで無効化。必要な楽器だけ有効にして作業。RAM/CPU節約に最も効果的な方法。右クリック→「トラックを無効化」。

Expression Maps: Art Conductor (BabylonWaves)

Spitfire、VSL、EW、Aaron Venture等の主要ライブラリ向けに事前構築されたExpression Maps集。手動で作るのに数週間かかるものが即使える。Cubase専用設計。

VEPro 7 — 必要か？

最初のテンプレートでは不要。 RAMが80-100GBを超えてプロジェクト読み込み/保存が問題になってから検討する。VEProの本質的価値は「1日に1回ロードするだけでいい」デカップリング機能。

テンプレート構築の推奨順序

  1. フォルダー構造とカラーコーディングを作成（楽器ゼロの状態で）
  2. グループトラックとVCA構造を作成
  3. FX Returnチャンネルを3〜5つ作成（リバーブ）
  4. コアスケルトン5楽器だけロード（Vln I, Vc, Brass Ens, Perc, Piano）
  5. ルーティング確認: 楽器→グループ→マスター→リバーブFX
  6. Expression Mapsを1楽器分だけ手動で設定して仕組みを理解
  7. Negative Track Delayを各楽器に設定
  8. テンプレートとして保存 → 32小節の曲を書いて検証
  9. 実際の制作で必要と判明したものだけ追加

共通音響空間の作り方（3ステップ）

Step 1: 両ライブラリをドライ状態に揃える

Spitfire Studio Strings: → Close mics: 70-80% → Tree mics: 20-30% → Ambient mics: 10-20%（または外部リバーブに任せてカット） → Outrigger mics: 必要に応じてSpitfireをワイドにしたい場合に追加 8dio Century Strings: → Close 1-3 を主に使用（Hall Micsは絞る） → Mixed A: フォーカスが必要な場合 → Mixed B / Wide: Spitfireの広がりに合わせたい場合に少量 → Hall micsをそのまま使いながら外部リバーブを重ねると二重空間になる

Step 2: 短いルームIRを8dioチャンネルに追加（ブレンドの秘訣）

Spitfire Studio Stringsにはドライ録音でも近接マイクに実際のAIR Studio 1のルーム感が微かに入っている。8dioのCloseマイクはより純粋にドライ。この差を埋めるため：

8dioチャンネルに非常に短いルームIRを追加: RT60: 0.3-0.5秒 Pre-delay: 50-80ms 目的: 近接場の「生きたルーム」感を付与

Step 3: 共有ホールリバーブバス（FXチャンネル）

Air Studios Reverb / LiquidSonics Cinematic Rooms / Valhalla Room RT60目標: 1.8〜2.3秒（Spitfire + 8dioのブレンド用） ※Lyndhurst Hall RT60 = 2.6秒なのでそれより少し短く Canopy height: Low または Medium（RT60短縮） Gallery Dampening: ON（吸収を追加、8dioのホール感と近づける） Early Level: 100%, Late Level: 15-25% Pre-delay（配置感）: ヴァイオリン : 15-25ms ヴィオラ : 20-30ms チェロ : 25-35ms コントラバス : 30-40ms Send量: Spitfire Studio Strings: -10〜-12 dB（ドライなので多め） 8dio Century Strings: -14〜-18 dB（すでに空間感あり）

ステレオ幅の管理

方法A: Spitfireを広げる

Waves S1 Stereo Enhancer、またはM/S処理でSideチャンネルを2-3 dBブースト。OutriggerマイクをSpitfireで上げる。

方法B: 8dioを絞る（推奨）

8dioのCloseマイクを中心に使い、Wide/Mixed Bを控えめに。M/S EQでSideを少し削る。

方法C: 深みで分離（最良）

Spitfire CloseをFront定義として使い、8dioのAmbient MicsをRoom Tailとして使う。「幅」ではなく「深み」で分離する自然な層構造。

Cubase FX Send設定（Valhalla Room 1インスタンスへ）: 弦楽 → Send Pre-delay: 0ms, Send量: -18dB（少なめ = 近い・前列） 木管 → Send Pre-delay: 5ms, Send量: -15dB 金管 → Send Pre-delay: 10ms, Send量: -12dB（多め = 遠い・後列） 打楽器→ Send Pre-delay: 15ms, Send量: -9dB （最も後列） ※ Valhalla Room本体のPre-delayはグローバル設定（25-35ms）にしておき、 各SendのPre-delayで微調整する。合計値が各セクションの「実効Pre-delay」になる。

PhoenixVerb — グローバルグルー設定

PhoenixVerbがグルーバスに最適な理由:
「Reverb Attack」パラメータを持つ（Valhalla Roomにはない）。これはリバーブテールが立ち上がるスピードを制御する。遅め設定にすると弦のアタックの後にリバーブがゆっくりブルームする — 澤野弘之の「弦が部屋に溶け込んでいく」あの効果。Bloomコントロール = 澤野サウンドの隠し鍵。
PhoenixVerb — Orchestral Master Bus グルー設定: Algorithm: Hall Reverb Time: 2.5 – 4.0s（澤野スタイルは長め） Pre-delay: 30 – 45ms（Valhallより長め → 全体的な距離感） Diffusion: 80 – 100%（スムーズなブレンド） Reverb Attack: 中〜遅め ← 澤野の「ブルーム」を作る核心パラメータ ER Level: 独立してコントロール（アーリーリフレクション量を別に設定可） High Cut: 6,000 – 8,000Hz Low Cut: 80 – 120Hz（低域のマッド防止 → Pro-Q3でも可） Mix（Insertの場合）: 12 – 20% Wet Mix（並列グルーバスの場合）: 100% Wet, リターン量 20%前後 澤野スタイル仕上げ: - Pro-Q3でPhoenixVerbバスの12kHz以上をハイシェルフカット （本物のホールは高域を吸収する。Air感過多 = ライブラリ感が出る） - Soothe2をValhalla Roomバスに挿してクレッシェンド時のハーシュネスを動的に抑制
